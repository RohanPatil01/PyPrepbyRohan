Data Types - Must Know
8 points
▼
Mutable: list, dict, set  |  Immutable: int, float, str, tuple, bool
list = ordered, mutable, allows duplicates  |  tuple = ordered, immutable
set = unordered, unique elements, O(1) lookup
dict = key-value, ordered (3.7+), keys must be hashable
bool is a subclass of int: True=1, False=0
None is a singleton: always compare with is None
Use isinstance(x, int) not type(x) == int: handles inheritance
Empty containers are falsy: [], {}, set(), '', 0, None all evaluate to False
📝
Strings - Key Points
6 points
▼
Strings are immutable: any modification creates a new string
Reverse: s[::-1]  |  Slice: s[start:stop:step]
join() is faster than += in loops: use ''.join(lst)
f-strings (3.6+) are fastest: f'Hello {name}'
split() → list  |  strip() → removes whitespace  |  replace() → new string
find() returns -1 if not found  |  index() raises ValueError
📋
Lists & Dicts - Critical Facts
8 points
▼
append(x) adds one item  |  extend([x,y]) adds many  |  insert(i,x) at index
pop() removes last  |  pop(0) removes first (O(n))  |  remove(x) by value
sort() in-place, returns None  |  sorted() returns new list
Shallow copy: a[:], a.copy(), list(a)  |  Deep copy: copy.deepcopy(a)
d.get(key, default): safe key access, never raises KeyError
d.setdefault(key, []): inserts if absent, returns existing if present
Merge dicts: {**d1, **d2} or d1 | d2 (3.9+)
Frequency count: from collections import Counter; Counter(lst)
⚙️
Functions - Essential Rules
7 points
▼
No return statement → returns None
Never use mutable default args: use def fn(x=None) then if x is None: x = []
*args = tuple of extra positional args  |  **kwargs = dict of keyword args
LEGB scope: Local → Enclosing → Global → Built-in
global x to modify global inside function  |  nonlocal x for enclosing scope
Lambda = anonymous, single-expression function. No statements allowed.
Closure = inner function that remembers enclosing scope variables
🏗️
OOP - Bare Minimum
6 points
▼
4 Pillars: Encapsulation, Abstraction, Inheritance, Polymorphism
__init__ = initialiser (runs after object created)  |  __new__ = creator
self = reference to current instance (not a keyword, just convention)
Instance method: takes self  |  Class method: @classmethod, takes cls  |  Static: @staticmethod
super().__init__() calls parent initialiser in child class
MRO (Method Resolution Order): Python uses C3 linearisation: view with ClassName.__mro__

Python Basics
5 points
▼
Python is interpreted, dynamically typed, garbage-collected
GIL (Global Interpreter Lock) allows only one thread to execute Python bytecode at a time
is = identity (same object)  |  == = equality (same value)
Integer caching: CPython caches -5 to 256: a is b may be True for small ints
Exponentiation is right-associative: 2 ** 3 ** 2 = 2 ** 9 = 512
🔒
Mutability & Copying
6 points
▼
Mutable: list, dict, set  |  Immutable: int, float, str, tuple, frozenset, bool
Assignment b = a creates an alias, not a copy: both point to same object
Shallow copy: new outer container, nested objects are shared
Deep copy (copy.deepcopy()): fully independent: all nested objects are new
Immutable objects are safe to use as dict keys & set elements (hashable)
For lists: += calls __iadd__ which mutates in-place; x = x + [1] creates a new list
a = [1, [2, 3]] b = a.copy() # shallow b[1].append(99) print(a) # [1, [2, 3, 99]] ← inner shared! import copy c = copy.deepcopy(a) # deep c[1].append(0) print(a) # [1, [2, 3, 99]] ← unchanged ✅
🔄
Comprehensions & Generators
5 points
▼
List comprehension: [x**2 for x in range(10) if x%2==0]
Dict comprehension: {k: v for k, v in d.items() if v > 0}
Generator expression (lazy): (x**2 for x in range(1000000)): no list in memory
Generator function: uses yield: pauses and resumes execution
Generators can only be iterated once: after exhaustion, they return nothing
🌐
Scope, Closures & Decorators
7 points
▼
LEGB: Local → Enclosing → Global → Built-in
If Python sees any assignment to a name in a function, it treats it as local throughout: even before the assignment (→ UnboundLocalError)
Use global x to write to global  |  nonlocal x to write to enclosing scope
Closure = inner function that captures variables from outer function's scope
Loop closure bug: all lambdas in a loop share the same loop variable: fix with lambda x, i=i: ...
Decorator = function that wraps another function. @dec is sugar for fn = dec(fn)
Always use @functools.wraps(func) inside decorators to preserve metadata
🏗️
OOP - Full Coverage
8 points
▼
Pillars: Encapsulation (data + methods), Abstraction (hide complexity), Inheritance (reuse), Polymorphism (same interface, different behaviour)
__new__ creates instance  |  __init__ initialises it: __new__ runs first
Class attribute (shared) vs instance attribute (per object) - a.x = 10 creates instance attr, shadowing class attr
@classmethod takes cls: used for factory methods  |  @staticmethod takes neither
@property lets you access a method like an attribute
Multiple inheritance uses MRO (C3 linearisation) - resolves diamond problem
Dunder methods: __str__, __repr__, __len__, __eq__, __lt__, __add__, __iter__
Duck typing: if it walks like a duck and quacks like a duck, it's a duck - Python checks behaviour, not type
🚨
Exceptions & File Handling
6 points
▼
try → except → else → finally: else runs only if no exception; finally always runs
Catch specific exceptions first, broad ones last. Avoid bare except:
Custom exception: class MyError(Exception): pass
Always use with open(): context manager guarantees file is closed even on error
Modes: 'r' read, 'w' write (overwrite), 'a' append, 'x' create new
CSV: csv.DictReader(f) gives each row as a dict keyed by column headers
